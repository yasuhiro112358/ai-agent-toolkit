# Explanation: なぜ一括投入型か

[diagram-keeper/](../index.md) > explanation


プロンプト全文 + マスタ + バンドルを 1 メッセージで投入する**一括投入型**を採用し、複数ターンに分けて貼り付ける対話型運用は採用しなかった。

---

## 一括投入の利点

- **品質最優先**: 1 回の AI 推論で全入力を突き合わせるため、抽出漏れ・整合性崩れのリスクが最も低い
- **運用シンプル**: 「全入力を揃えてから送信する」という単純なルールで運用できる

## 対話型を採用しない理由

| 問題 | 詳細 |
| --- | --- |
| 中間劣化リスク | 応答ごとに AI が中間要約・解釈・先走りを出しうる |
| 運用規律の負担 | 品質担保に「途中で余計な指示を差し込まない」等の規律が要る |

## 再検討のトリガー

- 1 回の貼り付けが UI 側で物理的に扱えないほど大きくなった
- 対話型でも品質を担保するガードプロンプトのベストプラクティスが確立された

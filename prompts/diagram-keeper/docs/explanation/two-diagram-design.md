# Explanation: なぜクラス図＋コールグラフの 2 枚構成か

単一の図に静的構造と呼び出し関係を混在させると粒度が混乱する（粒度混在アンチパターン）。2 枚に分離することで責務を明確にした。

---

## 2 枚それぞれが答える質問

| 図 | 質問に答える |
| --- | --- |
| クラス図 | 「このクラスはどんな構造をしているか」「どのクラスがどのクラスを継承しているか」 |
| コールグラフ | 「この API を変えたらどこに波及するか」「このメソッドはどこから呼ばれているか」 |

いずれも静的コード解析から確実に抽出できる要素（クラス定義・メソッド呼び出し関係）。コードが変わるたびに再生成可能であるため、マスタとして維持管理する意味がある。

## コールグラフとシーケンス図の違い

コールグラフは時系列を持たない静的な呼び出しマップ。「A が B を呼ぶ」という事実のみを記録し、「いつ」「どの条件で」は含めない。シーケンス図とは目的・読み方が明確に異なる。

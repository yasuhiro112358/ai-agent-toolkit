# 解説: なぜ要件定義書を中間層に置くか

要件定義書（IEEE 830 SRS-lite 形式）を中間層に置く理由は、「何をするか」と「なぜするか」を分離するためである。

| 問い | 担当する層 |
| ---- | --------- |
| なぜこのタスクが必要か | ゴール・アクター（要件定義書） |
| 何をするか | タスク概要・入力仕様・出力仕様（要件定義書） |
| どう指示するか | プロンプト（ステップ2の出力） |

この分離により、同じ要件定義書から異なるトーン・形式のプロンプトを生成することも可能になる。

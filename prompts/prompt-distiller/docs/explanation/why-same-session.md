# 解説: なぜ同一セッションで実行するか

[ドキュメント](../index.md) > なぜ同一セッションで実行するか

> **Diataxis: Explanation** — ステップ1を同一セッションで実行する設計の背景と意図。


ステップ1のプロンプトは、作業を行った **同じ会話セッション** の末尾に貼り付けて実行する設計になっている。「過去の会話をコピーして新しいセッションに貼り付ける」方式は採用しなかった。

## 同一セッション実行の利点

- コンテキストをコピーする手間がない（貼り付けるだけで即座に実行できる）
- 会話の全文が AI のコンテキストウィンドウに既に存在するため、抽出漏れが起きにくい
- 会話中のニュアンス（口調・前提・背景）がそのまま分析に反映される

## 「コンテキストを別途貼る」方式の問題

- コピーする範囲の判断が利用者に委ねられ、重要な文脈が抜け落ちるリスクがある
- UI によっては会話の全文コピーが難しい

## ステップ2でも同一セッション継続を推奨する理由

ステップ2（`prompts-generate.md`）も、ステップ1を実行した **同じ会話** の末尾にそのまま貼り付けられる。要件定義書は会話履歴に既に存在するため、コピー貼り付けが不要になる。

ただし、ステップ1の要件定義書をレビュー・修正してからステップ2に進む場合は、新しい会話に切り替えて要件定義書を貼り付ける方式（方法B）が適している。ステップ2はどちらの方式でも動作する。

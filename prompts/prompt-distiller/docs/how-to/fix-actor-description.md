# ハウツー: アクター記述を修正する

[ドキュメント](../index.md) > アクター記述を修正する

> **Diataxis: How-to** — アクターが抽象的な場合の修正手順。


## 症状

アクターが「エンジニア」「ユーザー」のような一言で終わっている。

## 修正

役割・専門知識レベル・利用シーンの3点を補足する。

```text
悪い例: エンジニア
良い例: バックエンドエンジニア（Python 経験3年以上）。コードレビュー依頼が集中する時間帯に、
         複数 PR を効率よく確認するために使う。
```

# ハウツー: 生成されたプロンプトを修正する

ステップ2で生成されたプロンプトを実際に使ってみて問題が見つかった場合、多くは要件定義書に修正の余地がある。症状から原因を特定して再実行する。

---

## プロンプトに固有値が残っている

**症状:** 生成されたプロンプトに特定のプロジェクト名・ファイル名・固有の数値が残っており、別の場面で使い回せない。

**原因:** 要件定義書の変数一覧で固有値が `[変数名]` に置き換えられていない。

**修正:** 要件定義書の変数一覧を見直し、残っている固有値をすべてプレースホルダーに置き換えてからステップ2を再実行する。

詳細: [変数化を修正する](fix-variable-extraction.md)

---

## タスクの説明が曖昧すぎる

**症状:** 生成されたプロンプトの「タスク」セクションが抽象的で、AI への指示として不十分。

**原因:** 要件定義書のタスク概要・出力仕様の記述が薄い。

**修正:** 要件定義書の以下のセクションを具体化してからステップ2を再実行する:

- タスク概要: 「何をするか」を一言で明確に
- 出力仕様: 構造・形式・分量・トーンを具体的に記述

---

## セクションが欠けている

**症状:** 生成されたプロンプトに必須セクション（用途・役割・タスク・入力形式・出力形式・制約条件）のいずれかが含まれていない。

**原因:** 要件定義書の対応するセクションが空または不十分。

**修正:** 要件定義書の不足セクションを補足してからステップ2を再実行する。

| 不足しているセクション | 要件定義書で確認するセクション |
| --------------------- | ----------------------------- |
| 用途 | ゴール・タスク概要 |
| あなたの役割 | アクター |
| 入力形式 | 入力仕様・変数一覧 |
| 出力形式 | 出力仕様 |
| 制約条件 | 制約条件 |

---

## トーン・形式が合わない

**症状:** 生成されたプロンプトの指示口調・出力形式（箇条書き/文章、Markdown/プレーンテキスト等）が意図と異なる。

**原因:** 要件定義書の制約条件に出力スタイルの指定がない。

**修正:** 要件定義書の制約条件に以下のようなスタイル指定を追加してからステップ2を再実行する:

```text
例:
- 出力は日本語で記述すること
- 箇条書きを使わず、文章形式で出力すること
- Markdown 形式（見出し・コードブロック）を使うこと
```

詳細: [暗黙の制約条件を補足する](fix-missing-constraints.md)

# ハウツー: 要件定義書の最終確認

[ドキュメント](../index.md) > 要件定義書の最終確認

> **Diataxis: How-to** — ステップ2に進む前の要件定義書レビューチェックリスト。


ステップ1で AI が生成した要件定義書は、AI の自己チェック・修正を経ているが、会話の文脈によっては精度が不十分な場合がある。ステップ2に進む前に人間がレビューすることで、最終的なプロンプトの品質が向上する。

## チェックリスト

- [ ] ゴールは「〜するために〜する」形式か
- [ ] アクターに役割・知識レベル・利用シーンが含まれているか
- [ ] タスク概要が一言で明確に表現されているか
- [ ] 入力仕様・出力仕様に形式・構造・トーンが具体的に書かれているか
- [ ] 制約条件に暗黙のルールも含まれているか
- [ ] 変数一覧の固有値がすべて `[変数名]` 形式になっているか

## 問題が見つかった場合

各問題の修正手順は以下を参照:

- [ゴール記述を修正する](fix-goal-statement.md)
- [アクター記述を修正する](fix-actor-description.md)
- [変数化を修正する](fix-variable-extraction.md)
- [暗黙の制約条件を補足する](fix-missing-constraints.md)

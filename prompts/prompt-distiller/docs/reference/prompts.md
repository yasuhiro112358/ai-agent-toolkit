# プロンプトファイル仕様

---

## requirements-extract.md（ステップ1）

### 概要

| 項目 | 内容 |
|------|------|
| 役割 | AI に「要件アナリスト」として振る舞わせる |
| 入力 | このファイルの全文（作業中の同一セッション末尾に貼り付ける） |
| 出力 | IEEE 830 SRS-lite 形式の要件定義書 |
| 挙動 | 作成 → 自己チェック → 問題があれば修正 → 出力 |

### 分析観点（7項目）

| 番号 | 観点 | 説明 |
|------|------|------|
| 1 | ゴール | なぜこのタスクが行われたか（「なぜした」の視点） |
| 2 | アクター | 誰がこのプロンプトを使うか。役割・知識レベル・利用シーン |
| 3 | タスク特定 | 何をしていたか（要約・翻訳・レビュー等）を一言で |
| 4 | 入力仕様 | ユーザーが提供した情報の種類・形式 |
| 5 | 出力仕様 | AI が生成した成果物の構造・形式・トーン |
| 6 | 制約抽出 | 明示的・暗黙的な制約条件を洗い出す |
| 7 | 変数化 | 固有値を `[変数名]` 形式のプレースホルダーに置き換える |

### 出力フォーマット（7セクション）

| セクション | 記述内容 |
|-----------|---------|
| ゴール | 「〜するために〜する」の形で一言 |
| アクター | 役割・専門知識レベル・利用シーンを記述 |
| タスク概要 | 何をするタスクか一言で |
| 入力仕様 | 情報の種類・形式。`[変数名]` を含む |
| 出力仕様 | 成果物の構造・形式・分量・トーン |
| 制約条件 | ルール・制限・禁止事項 |
| 変数一覧 | `[変数名]` と説明の対応表 |

### 自己チェック項目（6項目）

| チェック項目 | 確認内容 |
|------------|---------|
| ゴール | 「なぜするか」の視点で記述されているか |
| アクター | 役割・専門知識レベル・利用シーンが具体的か |
| タスク概要 | 何をするタスクか一言で明確に表現されているか |
| 入力仕様・出力仕様 | 形式・構造・トーンが具体的に特定されているか |
| 制約条件 | 明示的な制約だけでなく暗黙の制約も含まれているか |
| 変数一覧 | 固有値がすべて変数化され、プレースホルダーと説明が対になっているか |

## prompts-generate.md（ステップ2）

### 概要

| 項目 | 内容 |
| ---- | ---- |
| 役割 | AI に「プロンプトエンジニア」として振る舞わせる |
| 入力（同一会話） | このファイルの全文のみ（要件定義書は会話履歴から参照） |
| 入力（別会話） | このファイルの全文 + 末尾の `## 要件定義書` セクションに貼り付けた要件定義書 |
| 出力 | 汎用プロンプト（8セクション構成） |
| 挙動 | 作成 → 自己チェック → 問題があれば修正 → 出力 |

### 要件定義書からプロンプトへのマッピング

| プロンプトのセクション | 参照する要件定義書の項目 |
|----------------------|------------------------|
| 用途 | ゴール + タスク概要 |
| あなたの役割 | アクターが必要とする AI の専門性 |
| タスク | タスク概要 + 出力仕様 |
| 入力形式 | 入力仕様 + 変数一覧 |
| 出力形式 | 出力仕様 |
| 制約条件 | 制約条件 |
| 例（省略可） | 変数一覧から例値を当てはめて構成 |
| 注意事項 | 制約条件のうち運用上の注意に相当するもの |

### 出力フォーマット（8セクション）

| セクション | 記述内容 |
|-----------|---------|
| 用途 | このプロンプトで実現できることを一行で |
| あなたの役割 | AI に担わせる専門家・アシスタントとしての役割 |
| タスク | 具体的に何をするかの説明。手順がある場合は番号付きリスト |
| 入力形式 | ユーザーが提供すべき情報の種類と形式。`[変数名]` を含む |
| 出力形式 | 成果物の構造・形式・分量・トーン |
| 制約条件 | 守るべきルール・制限・品質基準 |
| 例（省略可） | Few-shot 例。入力と期待する出力のペアを1〜3組 |
| 注意事項 | よくある失敗パターンへの対処 |

### 自己チェック項目（5項目）

| チェック項目 | 確認内容 |
|------------|---------|
| 汎用性 | 固有データが変数化されており、別の場面でそのまま使えるか |
| 完結性 | プロンプト単体で AI が迷わず作業を開始できるか |
| 明確性 | 曖昧な表現がなく、期待する出力が具体的に記述されているか |
| セクション完備 | 用途・役割・タスク・入力形式・出力形式・制約条件がすべて埋まっているか |
| 変数名 | 変数名は意味が伝わる名前で記述されているか（日本語・英語どちらでも可） |

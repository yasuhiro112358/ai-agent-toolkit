# チュートリアル: 会話からプロンプトを蒸留する

このチュートリアルでは、生成 AI との会話をもとに汎用プロンプトを作るまでの全体の流れを体験する。

## 前提

- 対話型 AI（Claude.ai / ChatGPT / Gemini 等）にアクセスできる
- 汎用プロンプト化したい作業がその AI との会話で完了している

---

## ステップ1: 会話 → 要件定義書

### 1-1. プロンプトを会話の末尾に貼り付ける

作業が完了した **同じ会話** の入力欄に `prompts/requirements-extract.md` の全文を貼り付けて送信する。

AI はその会話の履歴を参照して分析するため、コンテキストを別途コピーして貼り付ける必要はない。

### 1-2. AI の出力を受け取る

AI が会話を分析し、以下の7セクションからなる要件定義書を出力する:

| セクション | 内容 |
| ---------- | ---- |
| ゴール | なぜこのタスクが行われたか（「なぜ」の視点） |
| アクター | 誰がこのプロンプトを使うか |
| タスク概要 | 何をするタスクか |
| 入力仕様 | ユーザーが提供する情報の種類・形式 |
| 出力仕様 | 成果物の構造・形式・トーン |
| 制約条件 | 守るべきルール・禁止事項 |
| 変数一覧 | 固有値を置き換えたプレースホルダー |

AI は出力後に自己チェックを行い、問題があれば修正してから最終版を返す。

### 1-3. 要件定義書をレビューする

出力された要件定義書を読んで、以下を確認する:

- ゴールが「なぜするか」の視点で書かれているか（「〜するために〜する」形式）
- アクターに役割・知識レベル・利用シーンが含まれているか
- 変数一覧の固有値が `[変数名]` 形式で適切に置き換えられているか

修正が必要であれば直接編集する。
詳しいレビュー方法は [要件定義書を改良する](../how-to/refine-requirements.md) を参照。

---

## ステップ2: 要件定義書 → 汎用プロンプト

### 2-1. プロンプトを貼り付ける

**方法A — 同じ会話で続けて実行する（推奨）:**

`prompts/prompts-generate.md` の全文を、ステップ1を実行した **同じ会話** の末尾に貼り付けて送信する。
要件定義書はすでに会話履歴にあるため、コピー貼り付けは不要。

**方法B — 別の会話で実行する:**

`prompts/prompts-generate.md` の全文をコピーし、末尾の `## 要件定義書` セクションにステップ1で得た要件定義書を貼り付けてから新しい会話に送信する。

### 2-2. AI の出力を受け取る

AI が要件定義書をもとに汎用プロンプトを出力する。出力には以下のセクションが含まれる:

| セクション | 内容 |
| ---------- | ---- |
| 用途 | このプロンプトで実現できることを一行で |
| あなたの役割 | AI に担わせる専門家としての役割 |
| タスク | 具体的に何をするか |
| 入力形式 | ユーザーが提供すべき情報 |
| 出力形式 | 成果物の構造・形式・分量 |
| 制約条件 | 守るべきルール |
| 例（省略可） | Few-shot 例 |
| 注意事項 | よくある失敗パターンへの対処 |

AI は出力後に自己チェックを行い、問題があれば修正してから最終版を返す。

### 2-3. 汎用プロンプトを保存する

出力されたプロンプトをファイルに保存する。命名例:

```text
summarize-document.md
review-code.md
translate-text.md
```

---

## 完了

会話から汎用プロンプトを作るまでの全体の流れを体験できた。

次にやること:

- 要件定義書をより精度高く仕上げる → [要件定義書を改良する](../how-to/refine-requirements.md)
- プロンプトファイルの仕様を確認する → [プロンプトファイル仕様](../reference/prompts.md)
- なぜこの設計なのかを理解する → [設計の判断](../explanation/design-decisions.md)

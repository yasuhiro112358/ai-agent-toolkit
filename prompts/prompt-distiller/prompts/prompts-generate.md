### セクションと要件定義書の対応

| プロンプトのセクション | 参照する要件定義書の項目 |
| ---------------------- | ------------------------ |
| 用途 | ゴール + タスク概要 |
| あなたの役割 | アクターが必要とする AI の専門性 |
| タスク | タスク概要 + 出力仕様 |
| 入力形式 | 入力仕様 + 変数一覧 |
| 出力形式 | 出力仕様 |
| 制約条件 | 制約条件 |
| 例（省略可） | 変数一覧から例値を当てはめて構成 |
| 注意事項 | 制約条件のうち運用上の注意に相当するもの |

### 禁止事項

- 要件定義書に記載されていない情報の創作・補完
- 特定の AI サービス・モデルに依存した機能の前提
- 固有データの変数化漏れ（汎用性を損なう記述）

### 自己チェックと修正

作成後、以下を確認してください。問題があれば修正してから出力してください：

- [ ] 固有データが変数化されており、別の場面でそのまま使えるか（汎用性）
- [ ] プロンプト単体で AI が迷わず作業を開始できるか（完結性）
- [ ] 曖昧な表現がなく、期待する出力が具体的に記述されているか（明確性）
- [ ] 用途・役割・タスク・入力形式・出力形式・制約条件がすべて埋まっているか
- [ ] 変数名は意味が伝わる名前で記述されているか（日本語・英語どちらでも可）

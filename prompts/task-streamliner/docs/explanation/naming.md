# プロダクト名「task-streamliner」の設計意図

## 命名の経緯

候補として検討した方向性は3つあった。

**1. 機能説明系**（`task-management` など）
— 説明的すぎて特徴が出ない。旧名称の問題点そのもの。

**2. 軽さ・携帯性系**（`paste-tasks`、`task-porter` など）
— 1ファイルのコピペという操作感は伝わるが、プロダクトの知的なイメージに欠ける。

**3. データ処理・クール系**（`task-streamer`、`task-dispatcher` など）
— タスクを分析してクールに捌くという体験を前面に出せる。

最終的に **`task-streamliner`** を採用した。

## 「streamliner」の語源と含意

`streamliner` は1930〜40年代の流線型高速列車の名称が語源。「無駄を削ぎ落とし、空気抵抗を最小化することで速く・美しく走る」という設計思想を体現した言葉だった。そこから `streamline`（効率化する・合理化する）という動詞が生まれ、現在は「プロセスをスリムにして本質だけ残す」という意味で広く使われている。

`task-streamliner` が表現するもの：

- タスクの混沌（Inbox の走り書き）を受け取り、分析・整理してクールに捌く
- 余分なツール・インフラを一切持ち込まない、最小限の構成
- Inbox の走り書きを受け取り、WBS + CPM で整理・順序付けして流す

## `task-streamer` を採用しなかった理由

`task-streamer` は語感が近かったが、`streamer` は現在「ライブ配信者（Twitch・YouTube）」の意味が支配的になっており、タスク管理ツールの名前として使うと混乱を招く可能性があった。`streamliner` はその問題がない。

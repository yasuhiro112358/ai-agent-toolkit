# 1ファイル構成の設計意図

## なぜ templates/tasks.md 1ファイルに統合したか

初期設計では `prompts/tasks-operate.md`（振る舞い定義）と `templates/tasks.md`（データ）の2ファイル構成だった。セッション開始時に両方をコピーしてチャットUIに貼り付ける必要があり、「どのファイルを貼るか」という判断コストと2回のコピペ操作が毎回発生していた。

`templates/tasks.md` の末尾に振る舞い定義を埋め込むことで、セッション開始の操作が1ファイルのコピペ1回に削減される。

## AI Instructions セクションの位置

ユーザーが日常的に参照するデータ（Inbox・WBS）が先に見えるよう、AI Instructions はファイル末尾に配置している。AIはファイル全体を読み込むため、末尾でも動作に影響しない。
